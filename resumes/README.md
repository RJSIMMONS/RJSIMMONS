# Résumé archive

Every résumé version is kept here, so older ones are never lost. The site always
links to **`/resume.pdf`** at the repo root — a copy of the current version — so the
public URL stays stable (`https://rjsimmons.github.io/RJSIMMONS/resume.pdf`).

## Naming convention

```
YYYY-MM-DD_Richard-Simmons-Resume_vXX.X.pdf
```

Example: `2026-06-21_Richard-Simmons-Resume_v26.6.pdf`

The date prefix keeps the folder sorted oldest → newest at a glance.

## Publishing a new version

1. Add the new PDF here using the naming convention above.
2. Copy it over the root `resume.pdf` (that's the one the site serves).
3. Commit and push.

(Just hand the new file to Claude and it'll do steps 1–3 for you.)
